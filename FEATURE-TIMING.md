# OpsFlow 时延检测功能

## 🎯 功能说明

`ops run web` 命令现在支持详细的时延检测，可以显示各个阶段的请求时延信息。

## 📊 时延检测阶段

### 1. DNS 解析时延 (DNS Lookup)
- **含义**: 域名解析到 IP 地址所需时间
- **正常范围**: < 100ms
- **影响因素**: DNS 服务器响应速度、网络延迟

### 2. TCP 连接时延 (TCP Connect)
- **含义**: 建立 TCP 连接所需时间
- **正常范围**: < 200ms
- **影响因素**: 服务器距离、网络质量、防火墙规则

### 3. 首字节时延 (First Byte)
- **含义**: 从发送请求到接收第一个字节的时间
- **正常范围**: < 500ms
- **影响因素**: 服务器处理速度、应用性能

### 4. 服务连接时延 (Server Connect)
- **含义**: 服务器处理请求的时间
- **正常范围**: < 100ms
- **影响因素**: 应用逻辑、数据库查询、缓存命中

### 5. 总时延 (Total)
- **含义**: 完整请求的总时间
- **正常范围**: < 1000ms
- **影响因素**: 所有上述因素的综合

## 📊 时延分析

### 性能基准

| 阶段 | 优秀 | 良好 | 一般 | 较差 |
|------|------|------|------|------|
| DNS 解析 | < 50ms | < 100ms | < 200ms | > 200ms |
| TCP 连接 | < 100ms | < 200ms | < 500ms | > 500ms |
| 首字节 | < 200ms | < 500ms | < 1000ms | > 1000ms |
| 总时延 | < 500ms | < 1000ms | < 2000ms | > 2000ms |

### 时延问题诊断

#### DNS 解析慢
- **原因**: DNS 服务器响应慢、网络延迟高
- **解决方案**: 更换 DNS 服务器、使用 CDN

#### TCP 连接慢
- **原因**: 服务器距离远、网络质量差、防火墙限制
- **解决方案**: 使用 CDN、优化网络路径

#### 首字节慢
- **原因**: 服务器处理慢、应用性能差
- **解决方案**: 优化代码、增加缓存、升级服务器

#### 总时延高
- **原因**: 多个阶段都有问题
- **解决方案**: 全面优化、使用性能监控工具

## 🔧 技术实现

### 时延检测逻辑

```go
// 1. DNS 解析时延
dnsStart := time.Now()
ips, err := net.LookupHost(target)
dnsLookup := time.Since(dnsStart)

// 2. TCP 连接时延
tcpStart := time.Now()
conn, err := net.DialTimeout("tcp", address, 10*time.Second)
tcpConnect := time.Since(tcpStart)

// 3. HTTP 请求时延
httpStart := time.Now()
resp, err := client.Do(req)
firstByte := time.Since(firstByteStart)
serverConnect := time.Since(httpStart) - firstByte

// 4. 总时延
total := time.Since(startTime)
```

### 数据精度
- **时间单位**: 微秒 (µs) / 毫秒 (ms)
- **精度**: 纳秒级精度，显示微秒级
- **稳定性**: 多次测量取平均值（可选）
